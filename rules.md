# 站点硬约束规则

本站是 linux.do 邀请链接互助分享站，以下规则用于审查领取者的申请理由是否合理。如果申请理由违反任何一条，请直接判定不合理（低分）。

## 拒绝条款

1. **本站不是 Linux 学习平台**：任何说"进来学 linux"、"想学 linux"、"学习 linux 命令"等以"学 linux"为主要目的的申请一律拒绝。linux.do 是一个综合性技术与生活社区，不是 Linux 教学社区。

2. **不准复读 / 刷字符**：纯重复字符（如打 50 个 1、aaaaa、复制粘贴同一句话凑字数）一律拒绝。

3. **不准 AI 生成**：如果申请文本明显由 AI 生成（套话排比、过度工整、空话连篇、缺乏具体细节、句式机械），直接拒绝。常见特征：
   - 大量"首先、其次、再次、最后"或"一方面…另一方面…"
   - 空洞的总分总结构
   - 反复强调"非常感谢"、"恳请通过"等套话
   - 提到"作为一名"、"在当今时代"、"希望能够"等机械模板
   - 无任何个人化、具体化的内容

4. **不准胡编乱造、胡言乱语**：申请理由必须与"为什么需要 linux.do 邀请"逻辑相关。无关内容、自相矛盾、明显瞎编（如"我是 Linus Torvalds 本人"）一律拒绝。

5. **必须具体**：申请理由要说明具体使用 linux.do 的目的，例如想看哪类话题、想参与哪类讨论、想分享什么内容。空泛的"想加入社区"、"想交朋友"分数不应过高。

## 评分指引

- **90-100**：理由具体、个人化、有真实使用场景说明，文风自然不像 AI。
- **75-89**：理由基本合理，有一定具体性，可放行。
- **50-74**：含糊、空泛、缺乏具体性，但没有明显违规。
- **0-49**：违反任一拒绝条款，或纯粹无意义内容。

只有 **score >= 75** 的申请才会获得邀请链接。
